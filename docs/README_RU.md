# Документация Documentor

Полная документация для библиотеки Documentor.

## Обзор

Documentor - это библиотека с открытым исходным кодом для предобработки документов перед RAG (Retrieval-Augmented Generation). Она извлекает структурированный контент из файлов PDF, DOCX и Markdown и преобразует их в унифицированную иерархическую структуру.

## Документация парсеров

### PDF Parser

**Подход**: Layout-based парсинг со специализированными процессорами

**Ключевые особенности**:
- Обнаружение layout через Dots.OCR для всех страниц
- Разные промпты для разных типов PDF:
  - Сканированные PDF: `prompt_layout_all_en` (layout + text + tables + formulas)
  - Текстовые PDF: `prompt_layout_only_en` (layout) + переобработка таблиц с `prompt_layout_all_en`
- Извлечение текста через PyMuPDF (для текстовых PDF) или из Dots OCR (для сканированных PDF)
- Автоматическое обнаружение сканированных документов
- Парсинг таблиц из HTML от Dots OCR
- Извлечение и хранение изображений
- Специализированные процессоры для модульности (layout, text, tables, images, hierarchy)

**Документация**: [PDF_PARSER.md](PDF_PARSER.md)

### DOCX Parser

**Подход**: Комбинированный подход (OCR layout + XML parsing + TOC validation)

**Ключевые особенности**:
- Обнаружение layout через Dots.OCR для Section-header и Caption
- XML парсинг для полного извлечения контента
- Парсинг оглавления для валидации заголовков
- Поиск пропущенных заголовков по правилам (адаптивные пороги, сопоставление свойств)
- Поиск подписей для таблиц и изображений из OCR
- Сопоставление структур таблиц (OCR vs XML) для валидации
- Автоматическое обнаружение сканированных документов с fallback на PdfParser
- Конвертация таблиц из XML в DataFrame
- Поддержка нумерованных заголовков с/без пробелов после номера
- Автоматическое определение и разделение элементов списка

**Документация**: [DOCX_PARSER.md](DOCX_PARSER.md)

### Markdown Parser

**Подход**: Regex-based парсинг (без LLM или OCR)

**Ключевые особенности**:
- Чистый regex-based парсинг
- Автоматическая конвертация таблиц в DataFrame
- Поддержка вложенных списков с правильной иерархией
- Построение иерархии заголовков
- Поддержка блоков кода, изображений, ссылок, цитат

**Документация**: [MARKDOWN_PARSER.md](MARKDOWN_PARSER.md)

## Архитектура

Все парсеры следуют унифицированной архитектуре:

1. **Вход**: LangChain `Document` с путем к файлу или контентом
2. **Определение формата**: Автоматическое определение формата
3. **Выбор парсера**: Выбирается соответствующий парсер
4. **Парсинг**: Документ парсится в структурированные элементы
5. **Построение иерархии**: Элементы организуются в иерархию с `parent_id`
6. **Выход**: `ParsedDocument` с унифицированной структурой

## Унифицированный формат вывода

Все парсеры возвращают одинаковую структуру `ParsedDocument`:

```python
ParsedDocument(
    source: str,
    format: DocumentFormat,
    elements: List[Element],
    metadata: Dict[str, Any]
)
```

Каждый `Element` содержит:
- `id`: Уникальный идентификатор
- `type`: ElementType (HEADER_1-6, TEXT, TABLE, IMAGE, и т.д.)
- `content`: Контент элемента
- `parent_id`: ID родительского элемента (для иерархии)
- `metadata`: Дополнительные метаданные (bbox, page_num, dataframe, и т.д.)

## Конфигурация

Вся конфигурация находится в `documentor/config/config.yaml`:

- `pdf_parser`: Настройки PDF парсера
- `docx_parser`: Настройки DOCX парсера
- `ocr_config.yaml`: Конфигурация OCR сервиса
- `llm_config.yaml`: Конфигурация LLM сервиса

## Пример использования

```python
from langchain_core.documents import Document
from documentor import Pipeline

# Инициализация пайплайна
pipeline = Pipeline()

# Создание документа
doc = Document(
    page_content="",
    metadata={"source": "path/to/document.pdf"}
)

# Парсинг документа
parsed_doc = pipeline.parse(doc)

# Доступ к элементам
for element in parsed_doc.elements:
    print(f"{element.type}: {element.content[:100]}")
```
