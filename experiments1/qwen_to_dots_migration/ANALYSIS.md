# Анализ использования Qwen в Documentor

## Текущее использование Qwen

### 1. OCR для извлечения текста (`ocr_text_with_qwen`)
**Где используется:**
- `documentor/processing/parsers/pdf/pdf_parser.py` → `_extract_text_by_bboxes()` (строка 820)
- Вызывается когда `use_ocr=True` (отсканированные PDF)

**Ситуация:**
- Используется для извлечения текста из изображений для категорий: Text, Section-header, Title, Caption
- Применяется когда PDF не имеет выделяемого текста (`is_text_extractable=False`)

**Проблема:**
- Dots OCR с промптом `prompt_layout_all_en` **уже извлекает текст** вместе с layout!
- Сейчас используется `prompt_layout_only_en` для layout, а потом Qwen для текста
- Это избыточно - можно использовать `prompt_layout_all_en` и получить все сразу

### 2. Парсинг таблиц (`parse_table_with_qwen`)
**Где используется:**
- `documentor/processing/parsers/pdf/pdf_parser.py` → `_process_tables()` (строка 1365)
- Используется для всех таблиц (не зависит от `use_ocr`)

**Ситуация:**
- Конвертирует изображение таблицы в markdown/dataframe
- Используется для всех таблиц, обнаруженных Dots OCR

**Проблема:**
- Dots OCR с `prompt_layout_all_en` **уже возвращает таблицы в HTML формате**!
- Можно парсить HTML напрямую, без Qwen

## Возможности Dots OCR

### Промпты в `ocr_config.yaml`:

1. **`prompt_layout_all_en`** - Layout + текст одновременно
   - Извлекает текст для всех категорий
   - Форматирование:
     - Table → HTML
     - Formula → LaTeX
     - Text, Title, etc. → Markdown
   
2. **`prompt_layout_only_en`** - Только layout без текста
   - Используется сейчас для layout detection
   - Потом Qwen используется для текста

3. **`prompt_ocr`** - Извлечение текста из изображения
   - Не используется в текущем коде

4. **`prompt_grounding_ocr`** - Извлечение текста по bbox
   - Не используется в текущем коде

## План миграции

### Этап 1: Замена OCR для текста
**Цель:** Использовать `prompt_layout_all_en` вместо `prompt_layout_only_en` + Qwen

**Изменения:**
1. В `layout_detector.py` изменить дефолтный промпт на `prompt_layout_all_en`
2. В `pdf_parser.py` убрать логику `use_ocr` для текста
3. Использовать текст напрямую из Dots OCR результатов
4. Удалить вызовы `ocr_text_with_qwen`

**Преимущества:**
- Один запрос вместо двух (layout + OCR)
- Быстрее и дешевле
- Меньше зависимостей

### Этап 2: Замена парсинга таблиц
**Цель:** Парсить HTML таблицы из Dots OCR вместо Qwen

**Изменения:**
1. Создать функцию `parse_table_from_html()` для конвертации HTML → markdown/dataframe
2. Заменить `parse_table_with_qwen()` на парсинг HTML из Dots OCR
3. Удалить `qwen_table_parser.py`

**Преимущества:**
- Не нужно рендерить таблицы в изображения
- Быстрее (нет OCR запроса для таблиц)
- Меньше зависимостей

### Этап 3: Очистка
**Цель:** Удалить весь код, связанный с Qwen

**Файлы для удаления:**
- `documentor/processing/parsers/pdf/ocr/qwen_ocr.py`
- `documentor/processing/parsers/pdf/ocr/qwen_table_parser.py`
- Секция `qwen_ocr` из `ocr_config.yaml`
- Импорты Qwen из всех файлов

**Файлы для изменения:**
- `documentor/processing/parsers/pdf/pdf_parser.py` - убрать логику Qwen
- `documentor/processing/parsers/pdf/ocr/layout_detector.py` - изменить промпт
- `documentor/processing/parsers/pdf/ocr/__init__.py` - убрать экспорты Qwen

## Риски и ограничения

### Потенциальные проблемы:
1. **Качество текста:** Dots OCR может давать менее точный текст, чем Qwen для некоторых документов
   - **Решение:** Протестировать на реальных документах, сравнить метрики

2. **HTML таблицы:** Dots OCR возвращает HTML, нужно корректно парсить
   - **Решение:** Использовать `beautifulsoup4` или `html.parser` для парсинга

3. **Формулы:** Dots OCR возвращает LaTeX, нужно убедиться, что это работает
   - **Решение:** Проверить обработку формул

4. **Обратная совместимость:** Если кто-то использует Qwen напрямую
   - **Решение:** Добавить deprecation warnings, потом удалить

## Метрики для проверки

После миграции нужно проверить:
1. **Точность текста:** CER/WER на тестовых документах
2. **Точность таблиц:** Корректность парсинга таблиц
3. **Производительность:** Время обработки документов
4. **Ресурсы:** Использование памяти и API запросов

**Dots OCR полностью заменяет Qwen** для всех задач:
- ✅ Извлечение текста (через `prompt_layout_all_en`)
- ✅ Парсинг таблиц (через HTML из Dots OCR)
- ✅ Определение layout (уже используется)
