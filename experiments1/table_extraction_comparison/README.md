# Эксперимент по сравнению методов извлечения таблиц из PDF

Этот эксперимент сравнивает два подхода к извлечению таблиц из PDF-документов:

1. **Метод 1 (`prompt_layout_all_en`)**: Использует Dots OCR с промптом `prompt_layout_all_en` для всех страниц. Один запрос на страницу, сразу получаем layout + текст (включая HTML таблиц).

2. **Метод 2 (`prompt_layout_only_en` + `prompt_layout_all_en` для таблиц)**: Использует Dots OCR с промптом `prompt_layout_only_en` сначала для определения layout. Если найдена таблица, то еще раз использует `prompt_layout_all_en` для этой таблицы (два запроса: один для layout, один для каждой таблицы).

## Структура директорий

```
experiments/table_extraction_comparison/
├── test_files/             # PDF файлы для обработки
├── results/                # Результаты обработки (JSON файлы с метриками)
├── pipeline.py             # Основной скрипт пайплайна
├── run_all.py              # Скрипт для пакетного запуска pipeline.py для всех PDF
├── setup_experiment.py     # Скрипт для инициализации структуры и копирования файлов
└── README.md               # Этот файл
```

## Подготовка к запуску

1. **Перейдите в директорию эксперимента:**
   ```bash
   cd experiments/table_extraction_comparison
   ```

2. **Создайте структуру директорий и скопируйте PDF файлы:**
   Запустите скрипт `setup_experiment.py` для создания необходимых папок и копирования тестовых PDF файлов из `experiments/metrics/test_files_for_metrics`.
   ```bash
   python setup_experiment.py
   ```

3. **Убедитесь, что у вас настроены переменные окружения для Dots OCR:**
   Убедитесь, что переменные окружения `DOTS_OCR_BASE_URL` и `DOTS_OCR_API_KEY` установлены.

## Запуск эксперимента

### Обработка одного PDF файла

Вы можете запустить `pipeline.py` для отдельного PDF файла:
```bash
python pipeline.py test_files/journal-10-67-5-721-729.pdf
```
Результаты будут сохранены в `results/journal-10-67-5-721-729_results.json`.

### Пакетная обработка всех PDF файлов

Для обработки всех PDF файлов в директории `test_files/` используйте `run_all.py`:
```bash
python run_all.py
```
Этот скрипт последовательно запустит `pipeline.py` для каждого PDF. Прогресс будет отображаться с помощью `tqdm`.

## Результаты

Для каждого обработанного PDF файла в директории `results/` будет создан JSON файл (`<pdf_name>_results.json`), содержащий:

- `pdf_file`: Имя исходного PDF файла.
- `timing`: Временные метрики:
  - `method1_seconds`: Время выполнения метода 1 (в секундах)
  - `method2_seconds`: Время выполнения метода 2 (в секундах)
  - `method1_minutes`: Время выполнения метода 1 (в минутах)
  - `method2_minutes`: Время выполнения метода 2 (в минутах)
  - `speedup`: Коэффициент ускорения (method2 / method1)
  - `time_saved_seconds`: Разница во времени (method2 - method1)
  - `time_saved_percent`: Процент экономии времени
- `tables_count`: Количество найденных таблиц:
  - `method1`: Количество таблиц в методе 1
  - `method2`: Количество таблиц в методе 2
- `method1`: Результаты метода 1:
  - `tables`: Список таблиц с их HTML содержимым
  - `timing_seconds`: Время выполнения
- `method2`: Результаты метода 2:
  - `tables`: Список таблиц с их HTML содержимым
  - `timing_seconds`: Время выполнения

### Метрики

**Измеряется только время выполнения** для каждого метода:
- Время обработки всех страниц
- Время на одну таблицу (если применимо)
- Общее время обработки документа

## Ожидаемые результаты

- **Метод 1** должен быть быстрее, если в документе много таблиц (один запрос на страницу вместо нескольких).
- **Метод 2** может быть быстрее, если в документе мало таблиц (не нужно извлекать текст для всех элементов, только для таблиц).

## Зависимости

- `PyMuPDF` (fitz) - для работы с PDF
- `Pillow` (PIL) - для работы с изображениями
- `tqdm` - для отображения прогресса
- `documentor` - основной пакет с интеграцией Dots OCR
