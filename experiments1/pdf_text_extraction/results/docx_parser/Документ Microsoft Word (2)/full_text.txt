Детектирование таблиц и их структуры:

Table Transformer (TATR)

Table Transformer (TATR) - это модель глубокого обучения, разработанная Microsoft Research для решения задач, связанных с таблицами в документах, в первую очередь для:

Обнаружения таблиц: Нахождения местоположения таблиц на странице документа (например, PDF, изображение).

Структурного анализа таблиц: Определения структуры таблицы, включая строки, столбцы, заголовки, ячейки и их взаимосвязи.

https://github.com/microsoft/table-transformer

https://huggingface.co/microsoft/table-transformer-detection

TF-ID (Table/Figure Identifier):

Эта модель применяет методы обнаружения объектов для автоматического определения и извлечения таблиц и рисунков из научных статей. Обученная на большом наборе данных с аннотациями, TF-ID способна распознавать визуальные шаблоны, характерные для таблиц и фигур, что значительно ускоряет процесс извлечения данных из научной литературы.

https://github.com/ai8hyf/TF-ID

CascadeTabNet

Эта модель использует каскадную архитектуру, объединяющую несколько стадий детекции для повышения точности обнаружения таблиц и их структур.

https://github.com/DevashishPrasad/CascadeTabNet

TableNet

Сеть, разработанная для одновременного обнаружения таблиц и их структурных элементов, таких как строки и столбцы, с использованием сверточных нейронных сетей.

https://github.com/jainammm/TableNet

GraphTSR (Graph-based Table Structure Recognition)

Подход, который представляет таблицу в виде графа и использует графовые нейронные сети для распознавания ее структуры.

https://github.com/ejlee95/Graph-based-TSR?tab=readme-ov-file_____________________________________________________________________________

1. Table Transformer (TATR)  (SOTA)

================================================================================
ID: 00000005
Type: TEXT
Parent: 00000004
================================================================================
Почему лучший:

Разработан Microsoft Research, использует архитектуру трансформеров, что обеспечивает высокую точность.

Работает с распознаванием структуры таблиц, ячеек, строк и заголовков.

Поддерживается на Hugging Face, что делает его легко доступным для использования и дообучения.

2. GraphTSR (Graph-based Table Structure Recognition)

================================================================================
ID: 00000007
Type: TEXT
Parent: 00000006
================================================================================
Почему в топе:

Использует Graph Neural Networks (GNN), что особенно полезно для сложных таблиц, где связи между ячейками не всегда линейны.

Обеспечивает гибкое структурное распознавание, хорошо подходит для нерегулярных таблиц.

Применим для анализа сложных научных документов.

3. TF-ID (Table/Figure Identifier)

================================================================================
ID: 00000009
Type: TEXT
Parent: 00000008
================================================================================
Преимущества:

Специализируется на детекции не только таблиц, но и фигур, что делает его универсальным инструментом для анализа научных статей.

Обучен на специализированных больших датасетах аннотированных документов.

Хорошо интегрируется с NLP-задачами (например, для поиска данных по таблицам в научных статьях).

5. TableNet

================================================================================
ID: 00000013
Type: TEXT
Parent: 00000012
================================================================================
Устаревшая архитектура:

Использует сверточные сети (CNN), которые уступают трансформерам и графовым нейронным сетям по качеству анализа структуры.

Хорош для простых таблиц, но плохо адаптируется к сложным макетам.

Менее точен в извлечении структуры (строки, заголовки, объединенные ячейки и т. д.).
