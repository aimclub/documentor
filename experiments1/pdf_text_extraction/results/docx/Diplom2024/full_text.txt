Рынок труда не стоит на месте: появляются новые профессии, а уже существующие профессии претерпевают изменения. Требования к знаниям, навыкам и умениям работников постоянно меняются. Для того, чтобы структурировать и унифицировать требования по профессиям, были разработаны профессиональные стандарты.

Профессиональный стандарт представляет собой характеристику квалификации, необходимой работнику для осуществления определенного вида профессиональной деятельности, в том числе выполнения определенной трудовой функции [1]. Другими словами, профессиональный стандарт является документом с набором необходимых требований для выполнения профессиональных обязанностей.

Существуют несколько сфер применения профессионального стандарта. Одной из них является сфера образования. Чтобы получить высококвалифицированного специалиста, его сначала нужно обучить таким образом, чтобы его знания и умения были востребованы. Для этого анализируются профессиональные стандарты, а затем, на основе результатов анализа, составляются профессиональные образовательные программы.

Цель: Разработать информационную систему для работы с информацией о профессиональных стандартах.

Задачи:

Реализовать просмотр информации о необходимых знаниях и умениях для каждой профессии;

Реализовать сравнение профессий по необходимым знаниям и умениям;

Реализовать поиск по необходимым знаниям и умениям;

Сравнить профессиональные стандарты;

Найти профессию оптимальную для направления ИСИТ.

1. Описание задачи

================================================================================
ID: 00000005
Type: TEXT
Parent: 00000004
================================================================================
В рамках работы необходимо было разработать информационную систему, которая представляет собой веб-приложение. Для веб-приложения требовалось реализовать следующий функционал:

Просмотр информации о необходимых знаниях и умениях для каждой профессии;

Сравнение профессий по необходимым знаниям и умениям;

Поиск по необходимым знаниям и умениям.

Для просмотра информации о необходимых знаниях и умениях для каждой профессии необходимо было реализовать вывод списка всех профессий, хранящихся в базе данных, и возможность выбора любой профессии из этого списка. При выборе определенной профессии требовалось отобразить необходимые знания и умения, относящиеся к данной профессии.

Для сравнения профессий по необходимым знаниям и умениям необходимо было реализовать выбор двух профессий для сравнения. Сравнение профессий нужно было осуществлять по формулировкам необходимых знаний и умений, относящихся к данным профессиям. Результатами сравнения должны были являться списки общих и схожих формулировок необходимых знаний и умений.

Для поиска по необходимым знаниям и умениям необходимо было реализовать ввод запроса, который далее должен был сравниваться с формулировками всех необходимых знаний и умений, хранящихся в базе данных. Результатами сравнения должны были быть списки необходимых знаний и умений, которые были признаны удовлетворительными для введенного запроса. Для каждого отобранного необходимого знания и умения требовалось вывести список профессий, к которым относится найденное необходимое знание или умение.

2. Используемые инструменты и языки

================================================================================
ID: 00000007
Type: TEXT
Parent: 00000006
================================================================================
Для разработки веб-приложения использовался язык программирования Python 3.9. Python позволяет вести веб-разработку со стороны сервера. В ходе разработки использовались следующие python-библиотеки:

re – для работы с регулярными выражениями;

sentence_transformers – для расчета косинусного расстояния и получения векторного представления;

pymorphy2 – для приведения слов к леммам;

torch – для получения векторного представления;

flashtext – для замены слов в предложении.

Для упрощения процесса разработки веб-приложения использовался бесплатный высокоуровневый python веб-фреймворк с открытым исходным кодом – django 4.2.7 [3].

Для создания структуры и оформления веб-страниц использовался язык разметки HTML5 и язык стилей CSS. Чтобы реализовать автозаполнение при вводе наименования профессии [4] использовался язык программирования JavaScript и его библиотека jQuery.

Для хранения информации о профессиональных стандартах использовалась свободно распространяемая реляционная система управления базами данных MySQL 8.0.31.

3.1. База данных

================================================================================
ID: 00000010
Type: TEXT
Parent: 00000009
================================================================================
Профессиональный стандарт имеет регистрационные номер, код, название и набор обобщенных трудовых функций. Каждая обобщенная трудовая функция имеет собственную характеристику. К характеристике относится уровень квалификации, список профессий и список трудовых функций. Каждая трудовая функция содержит списки трудовых действий, необходимых знаний и необходимых умений.

На рисунках 1-2 представлена инфологическая модель, описывающая устройство профессионального стандарта. Инфологическая модель содержит семь классов (сущностей): Профессиональный стандарт, Обобщенная трудовая функция, Профессия, Трудовая функция, Трудовое действие, Необходимое знание, Необходимое умение. Свойства, определенные у этих классов, представлены на рисунке 1. Связи, определенные между классами представлены на рисунке 2.

3.2. Архитектура веб-приложения

================================================================================
ID: 00000043
Type: TEXT
Parent: 00000042
================================================================================
Фреймворк django функционирует по архитектурному паттерну MVT. Структура MVT состоит из трех частей: M-Model (Модель), V-View (Представление), T-Template (Шаблон).

Модель определяет структуру хранимых данных, отвечает за получение и управление данными в базе данных. Обычно каждая модель сопоставляется с одной таблицей базы данных.

Представление получает и обрабатывает запрос от пользователя, а потом отправляет ответ пользователю. Если для формирования ответа требуется взаимодействие с базой данных, то представление обращается к необходимыми моделям. Для формирования ответа представление может использовать шаблон.

Шаблон определяет, как будут отображаться данные на веб-странице. Он содержит статические части желаемого вывода HTML, а также собственный синтаксис языка шаблонов django, описывающий, как будет вставлен динамический контент.

Созданные модели, представления и шаблоны для веб-приложения представлены в таблицах 13, 14, 15 соответственно.

3.3. Пользовательский интерфейс

================================================================================
ID: 00000054
Type: TEXT
Parent: 00000053
================================================================================
Сначала пользователь попадает на главную страницу. Через меню, которое содержится на всех страницах веб-приложения, пользователь может перейти на следующие страницы:

Главная страница;

Страница со списком всех профессий;

Страница для сравнения профессий по необходимым знаниям и умениям;

Страница для поиска по необходимым знаниям и умениям.

Для возвращения на главную страницу пользователю нужно нажать на название веб-приложения в меню.

Для перехода на страницу со списком всех профессий, пользователю необходимо нажать на пункт меню «Информация о профессиях». С данной страницы пользователь может перейти на страницу c информацией о необходимых знаниях и умениях для профессии, выбрав определенную профессию из списка.

Чтобы перейти на страницу для сравнения профессий по необходимым знаниям и умениям, пользователю необходимо нажать на пункт меню «Сравнение профессий». Заполнив верно на данной странице форму для сравнения профессий, пользователь может перейти на страницу с результатами сравнения, нажав на кнопку «Сравнить».

Для осуществления перехода на страницу для поиска по необходимым знаниям и умениям, пользователю необходимо нажать на пункт меню «Поиск по знаниям и умениям». Введя в поле для ввода запрос и нажав на кнопку «Найти похожие», пользователь может перейти на страницу с результатами поиска по необходимым знаниям и умениям.

3.4. Просмотр информации о профессиях

================================================================================
ID: 00000086
Type: TEXT
Parent: 00000085
================================================================================
К информации о профессиях относятся необходимые знания и необходимые умения. Их необходимо было получить из базы данных. Для этого была создана функция get_knowledge_skills(profession_name: str, with_tensor:bool).

Функция принимает на вход два параметра. Первый параметр profession_name - название профессии, для которой необходимо получить необходимые знания и умения. Второй параметр with_tensor является указателем на то, нужно ли получать векторное представление для каждого найденного необходимого знания и умения.

Если значение параметра with_tensor равно False, то функция возвращает два списка. Первый список содержит все необходимые знания, относящиеся к профессии profession_name, а второй - все необходимые умения, относящиеся к профессии profession_name.

Если значение параметра with_tensor равно True, то функция возвращает два словаря. Первый словарь содержит все необходимые знания, относящиеся к профессии profession_name, а второй - все необходимые умения, относящиеся к профессии profession_name. У этих двух словарей ключом является формулировка необходимого знания или умения, а значением - векторное представления этой формулировки.

В данном случае векторное представление не нужно, поэтому значение параметра with_tensor равно False.

3.5. Сравнение профессий

================================================================================
ID: 00000094
Type: TEXT
Parent: 00000093
================================================================================
Сравнение двух профессий осуществляется по формулировкам необходимых знаний и умений, относящихся к данным профессиям. Для двух профессий находятся общие и схожие формулировки. К общим формулировкам относятся те формулировки, которые полностью совпадают по написанию. К схожим формулировкам относятся те формулировки, которые отличаются по написанию, но являются похожими или подобными по смыслу. Примеры схожих формулировок представлены в таблице 18.

Для проверки того, являются ли формулировки общими используется посимвольное сравнение строк (строки должны совпасть).

Для определения того, являются ли формулировки схожими по смыслу используются векторные представления (эмбеддинги). Векторное представление (эмбеддинг) – это числовой вектор, который получается в результате преобразования текста. Данный вектор отражает семантическую и синтаксическую информацию о данных. Для каждой формулировки необходимого знания и умения векторное представление было получено с помощью модели нейронной сети – sentence-transformers/paraphrase- multilingual-MiniLM-L12-v2 [5]. Вектора, полученные с помощью данной модели имеют длину 384, то есть состоят из 384 чисел.

Об отборе схожих формулировок на основе векторных представлений описано в подразделе «Описание алгоритма сравнения профессий».

Описание алгоритма сравнения профессий

================================================================================
ID: 00000099
Type: TEXT
Parent: 00000098

Алгоритм сравнения профессий начинает свою работу с получения данных о необходимых знаниях и умениях для каждой профессии. Под данными о необходимых знаниях (умениях) подразумевается формулировка необходимого знания (умения) и ее векторное представление.

После получения для двух профессий наборов данных про необходимые знания (умения) среди этих наборов находятся общие формулировки необходимых знаний (умений). Найденные общие формулировки заносятся в отдельный список. Из двух наборов данные об общих формулировках необходимых знаний (умений) отбрасываются.

Далее начинается поиск схожих формулировок. Для начала составляется список всех возможных пар для сравнения. Из этого списка убираются те пары формулировок, для которых косинусное расстояние между их векторными представлениями, округленное до двух знаков после запятой, меньше, чем 0,85. После этого для каждой формулировки из пары формируется список токенов. Для формирования списка токенов формулировка разбивается на отдельные слова, где каждое слово приводится к начальной форме. Затем из получившегося списка токенов убираются стоп-слова:

Затем среди получившихся списков токенов находится наибольший и наименьших из списков и осуществляется проверка на то, является ли множество токенов меньшего списка подмножеством множества токенов большего списка. Если результат положительный (является подмножеством), тогда формулировки признаются схожими, они добавляются в список схожих, и берется следующая пара для сравнения. Если результат отрицательный (не является подмножеством), тогда формулировки не признаются схожими и берется следующая пара для сравнения.

Когда все возможные пары формулировок были сравнены, работа алгоритма завершается возвратом списка общих и списка схожих формулировок

Реализация алгоритма сравнения профессий

Для реализации алгоритма сравнения профессий был создан класс ComparisonOfFormulations. В данном классе были реализованы следующие методы:

compare_formulations(self, first_forms: list[dict], second_forms: list[dict]);

cos_score(vector1: Tensor, vector2: Tensor);

get_tokens(self, formulation: str);

__replace_keywords(self, tokens: list[str]);

subset_check(self, first_tokens: list[str], second_tokens: list[str]).

Метод compare_formulations(self, first_forms: list[dict], second_forms: list[dict]) предназначен для нахождения общих и схожих формулировок. На вход он принимает ссылку на экземпляр класса (self), список формулировок необходимых знаний или умений для первой профессии (first_forms) и список формулировок необходимых знаний или умений для второй профессии (second_forms). Возвращает метод множество общих формулировок и список пар схожих формулировок.

Метод cos_score(vector1: Tensor, vector2: Tensor) предназначен для нахождения косинусного расстояния. Входными данными для метода являются два векторных представления (vector1 и vector2). Результатом работы метода является округленное до двух знаков после запятой значение косинусного расстояния.

Метод get_tokens(self, formulation: str) предназначен для формирования списка токенов для формулировки необходимого знания (умения). На вход принимает ссылку на экземпляр класса (self) и формулировку (formulation), а возвращает список токенов.

Метод __replace_keywords(self, tokens: list[str]) предназначен для поиска и замены ключевых токенов в списке токенов. В качестве входных данных метод принимает ссылку на экземпляр класса (self) и список токенов (tokens). Возвращает метод преобразованный список токенов.

Метод subset_check(self, first_tokens: list[str], second_tokens: list[str]) предназначен для осуществления проверки двух списков токенов на подмножество. На вход он принимает ссылку на экземпляр класса (self) и два списка токенов (first_tokens и second_tokens), а возвращает значение True, если один из списков токенов оказался подмножеством второго списка, иначе - False.

При реализации также использовалась ранее описанная функция get_knowledge_skills(profession_name: str, with_tensor: bool) для получения списка необходимых знаний (умений) и их векторных представлений. Для получения векторного представления из строки с векторными представлением была создана функция get_tensor_from_str(tensor_string: str). Функция на вход получает строку с векторным представлением (tensor_string), а возвращает векторное представление.

Из средств других библиотек языка python использовались:

Функция findall() библиотеки re – для получения списка всех слов из формулировки;

Функция util.cos_sim() фреймворка sentence_transformers – для нахождения векторного расстояния;

Класс KeywordProcessor библиотеки flashtext и его методы add_keywords_from_dict() и replace_keywords() – для добавления словаря Synonyms и для замены ключевых токенов;

Класс MorphAnalyzer библиотеки pymorphy2 и его метод normal_forms() – для получения начальной формы слов.

3.6. Поиск по знаниям и умениям

================================================================================
ID: 00000120
Type: HEADER_3
Parent: 00000119
================================================================================
Описание алгоритма поиска по знаниям и умениям

================================================================================
ID: 00000121
Type: TEXT
Parent: 00000120
================================================================================
Алгоритм поиска по знаниям и умениям начинает свою работу с извлечения данных о всех необходимых знаниях и умениях, хранящихся в базе данных. Под данными подразумевается формулировки необходимых знаний или умений и их векторные представления.

После получения данных для запроса находится векторное представление и список токенов. Далее с использованием векторных представлений проводится первичный отбор формулировок, которые удовлетворяли бы поисковому запросу. Среди всех необходимых знаний и умений отбрасываются те, для которых округленное до 2 знаков после запятой косинусное расстояние между векторными представлениями формулировки и запроса оказалось меньше 0,5.

Для оставшихся формулировок необходимых знаний и умений формируются списки токенов. Списки токенов для запроса и для формулировок необходимых знаний и умений преобразуются за счет замены ключевых токенов из словаря Synonyms (Таблица 15).

Далее начинается второй этап отбора формулировок, удовлетворяющих поисковому запросу. Ранее полученный список токенов запроса сравнивается со списками токенов необходимых знаний и умений. Сначала осуществляется поиск более длинного и более короткого списков, а затем осуществляется проверка на подмножество. Те формулировки, для которых проверка на подмножество оказалась отрицательной – отбрасываются. Оставшиеся формулировки необходимых знаний и умений признаются удовлетворительными для введенного поискового запроса. На возврате оставшихся формулировок необходимых знаний и умений работа алгоритма поиска заканчивается.

Реализация алгоритма поиска по знаниям и умениям

================================================================================
ID: 00000123
Type: TEXT
Parent: 00000122
================================================================================
Для реализации алгоритма поиска по знаниям и умениям использовалась ранее описанная функция get_tensor_from_str(tensor_string: str), а также ранее описанные методы класса ComparisonOfFormulations:

cos_score(vector1: Tensor, vector2: Tensor);

get_tokens(self, formulation: str);

__replace_keywords(self, tokens: list[str]);

subset_check(self, first_tokens: list[str], second_tokens: list[str]).

Помимо использования данных методов, в класс ComparisonOfFormulations был добавлен метод search_by_formulations(self, query: str, knowledge: dict, skills: dict), а также была создана новая функция build_dict_prof_nec(similar_necessaries: str, is_for_skills: bool).

Метод search_by_formulations(self, query: str, knowledge: dict, skills: dict) предназначен для нахождения необходимых знаний и умений, которые удовлетворяли бы поисковому запросу. Метод на вход принимает четыре параметра. Первый параметр self является ссылкой на экземпляр класса, второй параметр query – формулировкой поискового запроса, третий и четвертый параметры knowledge и skills – данными о необходимых знаниях и умениях. Возвращает метод два списка. Первый список содержит необходимые знания, удовлетворяющие поисковому запросу, а второй список – необходимые умения, удовлетворяющие поисковому запросу.

Функция build_dict_prof_nec(similar_necessaries: str, is_for_skills: bool) предназначена для получения списка профессий, к которым относится необходимое знание или умение. На вход она принимает формулировку необходимого знания или умения и указатель на то, является ли переданная формулировка необходимым умением. Если значение True, тогда переданная формулировка является необходимым умением, иначе – False. Результатом работы функции является словарь с ключами «form» и «profs». Первый ключ хранит значение формулировки, а второй ключ - список профессий.

Для получения векторного представления запроса используется модель «sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2» [5] и метод encode() класса SentenceTransformer фреймворка sentence_transformers.

4.2. Поиск профессии, наиболее соответствующей компетенциям ИСИТ

================================================================================
ID: 00000143
Type: TEXT
Parent: 00000142
================================================================================
В рамках ВКР был осуществлен поиск профессии наиболее соответствующей компетенциям подготовки по направлению «Информационные системы и технологии» (ИСИТ) с использованием разработанного алгоритма поиска по знаниям и умениям.

Источником компетенций ИСИТ служил документ «Основная профессиональная образовательная программа подготовки бакалавра по направлению 09.03.02 (2020-2024 годы обучения)» [2]. Из данного документа было получено 99 компетенций. Из них – 39 универсальных компетенций, 42 общепрофессиональных компетенций и 18 профессиональных компетенций.

Для поиска наиболее соответствующей профессии использовалась функция search_by_formulations(), которой в качестве поискового запроса передавалась одна и компетенций ИСИТ. В результате прогона всех компетенций через функцию search_by_formulations() получились списки необходимых знаний и умений, которые удовлетворяют компетенциям ИСИТ. Затем для каждого необходимого знания и умения находились профессии, к которым относятся данные необходимые знания и умения. В заключении для каждой найденной профессии был осуществлен подсчет того, сколько данная профессия встречалась.

В результате получилось, что несколько профессий являются наиболее соответствующими компетенциям подготовки по направлению ИСИТ. К ним относятся следующие профессии:

Ведущий специалист по внедрению ИС;

Программист-проектировщик ИС;

Ведущий консультант по ИС;

Бизнес-аналитик;

Руководитель сервисной службы по ИС.

Данные профессии относятся к профессиональному стандарту 06.015 «Специалист по информационным системам». На основе данного стандарта были сформулированы профессиональные компетенции для направления ИСИТ.

В результате проделанной работы было разработано веб-приложение для работы с информацией о профессиональных стандартах. Веб-приложение было разработано на языке программирования Python с использованием фреймворка django.

В получившемся веб-приложении возможно получить для 246 профессий информацию о необходимых знаниях и умениях, сравнивать профессии по необходимым знаниям и умениям, а также осуществлять поиск по необходимым знаниям и умениям.

Для сравнения профессий по необходимым знаниям и умениям в веб-приложении был разработан алгоритм, который находит общие и схожие формулировки необходимых знаний и умений для двух профессий.

Для поиска по необходимым знаниям и умения в веб-приложении был разработан алгоритм, который находит необходимые знания и умения, удовлетворяющие поисковому запросу.
