# DOTS OCR Pipeline - Важные моменты

## Извлечение текста: DOTS OCR координаты + PyMuPDF текст

### Ключевая идея

**DOTS OCR используется ТОЛЬКО для определения layout и координат блоков, НЕ для OCR распознавания текста.**

### Процесс:

1. **DOTS OCR** определяет структуру документа:
   - Находит координаты текстовых блоков
   - Определяет типы элементов (Text, Title, Table, Picture, Caption, etc.)
   - Возвращает bounding boxes для каждого элемента

2. **PyMuPDF (fitz)** извлекает текст:
   - Использует координаты, полученные от DOTS OCR
   - Извлекает текст напрямую из PDF (если текст доступен в PDF)
   - НЕ использует OCR для распознавания текста

### Преимущества:

- **Скорость**: PyMuPDF извлекает текст быстрее, чем OCR
- **Точность**: Текст из PDF обычно точнее, чем OCR распознавание
- **Экономия ресурсов**: Не нужно запускать OCR LLM для распознавания текста
- **Структура**: DOTS OCR обеспечивает правильную структуру и координаты

### Когда использовать OCR:

OCR (например, через DOTS OCR или другой OCR движок) нужен ТОЛЬКО когда:
- PDF содержит отсканированные изображения (нет текстового слоя)
- Текст в PDF недоступен или поврежден
- Нужно распознать текст на изображениях/рисунках

### Пайплайн:

```
PDF → DOTS OCR (layout detection) → Координаты блоков
                                    ↓
PDF → PyMuPDF (text extraction) → Текст по координатам
                                    ↓
Объединение → Структурированный документ с текстом и координатами
```

### Реализация:

1. Запускаем DOTS OCR на странице PDF → получаем JSON с координатами
2. Для каждого текстового блока:
   - Используем координаты из DOTS OCR
   - Извлекаем текст из PDF через PyMuPDF по этим координатам
   - Объединяем координаты и текст

### Пример структуры данных:

```json
{
  "elements": [
    {
      "category": "Text",
      "bbox": [100, 200, 500, 300],
      "text": "Текст, извлеченный через PyMuPDF по координатам [100, 200, 500, 300]"
    },
    {
      "category": "Title",
      "bbox": [100, 50, 500, 100],
      "text": "Заголовок, извлеченный через PyMuPDF"
    }
  ]
}
```

### Важно:

- **НЕ парсим текст через OCR LLM**, если текст доступен в PDF
- DOTS OCR используется только для **layout detection** (определение структуры)
- PyMuPDF используется для **text extraction** (извлечение текста)
