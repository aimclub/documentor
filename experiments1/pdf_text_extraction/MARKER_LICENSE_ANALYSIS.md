# Анализ лицензионной совместимости с Marker

## Текущая ситуация

### Лицензии проектов

**Marker:**
- **Код:** GPL-3.0 (GNU General Public License версия 3)
- **Модели:** AI Pubs Open Rail-M License (Modified)
- Автор: Endless Labs, Inc. (Copyright 2024)

**Наш проект (documentor):**
- **Лицензия:** MIT
- Автор: angrymuskrat

### Проблема совместимости

**GPL-3.0** - это **copyleft** лицензия, что означает:
- ✅ Можно использовать код
- ✅ Можно модифицировать код
- ✅ Можно распространять код
- ❌ **НО:** Если вы используете GPL код в своем проекте, ваш проект **тоже должен быть под GPL** (или совместимой лицензией)

**MIT** - это **permissive** лицензия:
- ✅ Можно использовать, модифицировать, распространять
- ✅ Можно использовать в коммерческих проектах
- ✅ Можно использовать в проприетарных проектах
- ❌ **НО:** Несовместима с GPL в обратном направлении

## Что можно делать

### ✅ Разрешено

1. **Использовать marker как зависимость (уже делаем)**
   - Установка через `pip install marker-pdf`
   - Использование через API/CLI
   - Это нормально, так как мы не включаем код marker в наш проект

2. **Вдохновляться идеями и алгоритмами**
   - Изучать код для понимания подходов
   - Переписывать функциональность с нуля
   - Использовать те же библиотеки (surya, pdftext)

3. **Использовать модели marker**
   - Модели под AI Pubs Open Rail-M License
   - Более свободная лицензия для моделей
   - Можно использовать в коммерческих проектах (с ограничениями по выручке <$2M)

### ❌ Запрещено

1. **Прямое копирование кода из marker**
   - Копирование функций, классов, методов
   - Включение файлов из marker в наш проект
   - Это потребует изменить лицензию проекта на GPL

2. **Использование кода marker без указания лицензии**
   - Нарушение авторских прав
   - Юридические проблемы

## Рекомендации

### Вариант 1: Использовать как зависимость (рекомендуется)
**Текущий подход - правильный:**
- Используем marker через `pip install marker-pdf`
- Вызываем через API или CLI
- Не включаем код marker в наш проект

**Плюсы:**
- ✅ Полная совместимость с MIT лицензией
- ✅ Нет юридических проблем
- ✅ Получаем обновления marker автоматически

**Минусы:**
- ❌ Зависим от внешней библиотеки
- ❌ Не можем напрямую модифицировать код marker

### Вариант 2: Переписать функциональность
**Если нужна собственная реализация:**

1. **Изучить алгоритмы marker:**
   - Как парсятся таблицы (см. `MARKER_TABLE_PARSING.md`)
   - Как обрабатывается текст
   - Как работает OCR pipeline

2. **Переписать с нуля:**
   - Использовать те же библиотеки (surya, pdftext)
   - Реализовать похожую логику, но свой код
   - Можно использовать те же модели

3. **Указать источник вдохновения:**
   - В комментариях: "Inspired by marker's approach"
   - В README: "Algorithm inspired by marker library"

**Плюсы:**
- ✅ Полный контроль над кодом
- ✅ Можно оптимизировать под наши нужды
- ✅ Нет зависимости от внешней библиотеки

**Минусы:**
- ❌ Нужно время на разработку
- ❌ Нужно поддерживать код самостоятельно

### Вариант 3: Отдельный GPL модуль
**Если очень нужен код marker:**

1. Создать отдельный модуль под GPL
2. Использовать его как зависимость
3. Основной проект остается MIT

**Плюсы:**
- ✅ Можно использовать код marker напрямую
- ✅ Основной проект остается MIT

**Минусы:**
- ❌ Сложная структура проекта
- ❌ Два разных лицензионных режима

### Вариант 4: Связаться с авторами
**Для коммерческого использования:**

- Связаться с Endless Labs, Inc.
- Запросить коммерческую лицензию
- Убрать GPL требования

**Ссылки:**
