# Пайплайн обработки всех изображений через DOTS OCR

## Описание

Скрипт `process_all_images_dots_ocr.py` обрабатывает все изображения из директории `results/dots_ocr_test/temp_images` через DOTS OCR API.

## Функциональность

1. **Увеличение изображений**: Каждое изображение увеличивается в 2 раза перед отправкой в DOTS OCR
2. **Отправка в DOTS OCR**: Увеличенные изображения отправляются в DOTS OCR API для получения layout информации
3. **Отрисовка bbox**: Bbox отрисовываются на увеличенном изображении (bbox уже в координатах увеличенного изображения)
4. **Сохранение результатов**:
   - Изображения с отрисованными bbox сохраняются в `results/dots_ocr_test/processed_images/`
   - JSON файлы с bbox данными сохраняются в `results/dots_ocr_test/bbox_results/`
   - Сводка обработки сохраняется в `results/dots_ocr_test/processing_summary.json`

## Важные моменты

### Увеличение изображений

- Изображения увеличиваются в 2 раза перед отправкой в DOTS OCR
- Bbox возвращаются в координатах **увеличенного** изображения
- При отрисовке bbox используются координаты увеличенного изображения без дополнительного масштабирования

### Формат bbox

Bbox возвращаются в формате `[x1, y1, x2, y2]` в координатах увеличенного изображения.

## Использование

### Базовое использование

```bash
cd experiments/pdf_text_extraction
python process_all_images_dots_ocr.py
```

### Переменные окружения

Скрипт использует следующие переменные окружения (с дефолтными значениями):

- `DOTS_OCR_BASE_URL` - URL API (по умолчанию: `http://10.32.2.11:8069/v1`)
- `DOTS_OCR_API_KEY` - API ключ (по умолчанию: `security-token-abc123`)
- `DOTS_OCR_MODEL_NAME` - Имя модели (по умолчанию: `/nfshome/avkhauhia/DotsOCR`)
- `DOTS_OCR_TEMPERATURE` - Температура (по умолчанию: `0.1`)
- `DOTS_OCR_MAX_TOKENS` - Максимальное количество токенов (по умолчанию: `65000`)
- `DOTS_OCR_TIMEOUT` - Таймаут запроса в секундах (по умолчанию: `30`)

### Пример с переменными окружения

```bash
export DOTS_OCR_BASE_URL="http://10.32.2.11:8069/v1"
export DOTS_OCR_API_KEY="your-api-key"
export DOTS_OCR_MODEL_NAME="/nfshome/avkhauhia/DotsOCR"

python process_all_images_dots_ocr.py
```

## Структура выходных файлов

### Обработанные изображения

```
results/dots_ocr_test/processed_images/
├── 2304.05128v2_page_1_with_bbox.png
├── 2304.05128v2_page_2_with_bbox.png
└── ...
```

### JSON файлы с bbox

```
results/dots_ocr_test/bbox_results/
├── 2304.05128v2_page_1_bbox_results.json
├── 2304.05128v2_page_2_bbox_results.json
└── ...
```

### Формат JSON файла

```json
[
  {
    "category": "Text",
    "bbox": [0, 0, 1007, 1007]
  },
  {
    "category": "Picture",
    "bbox": [100, 200, 500, 600]
  }
]
```

### Сводка обработки

Файл `processing_summary.json` содержит:
- Общее количество изображений
- Количество успешно обработанных
- Количество ошибок
- Детальную информацию по каждому изображению

## Зависимости

- `PIL` (Pillow) - для работы с изображениями
- `openai` - для работы с DOTS OCR API
- `tqdm` (опционально) - для отображения прогресс-бара
- `dots_ocr` - для отрисовки bbox (должен быть в `dots.ocr/`)

## Обработка ошибок

Скрипт продолжает обработку даже при ошибках на отдельных изображениях. Все ошибки логируются и сохраняются в сводке.

## Производительность

- Обработка одного изображения занимает примерно 5-15 секунд (зависит от размера и сложности)
- Для 78 изображений общее время обработки составит примерно 6-20 минут
