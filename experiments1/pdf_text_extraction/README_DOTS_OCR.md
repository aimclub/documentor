# Тестирование DOTS OCR

## Проблема с зависимостями

Установка DOTS OCR может вызвать конфликты с основными зависимостями проекта:
- `numpy`: требуется <2.0.0, но для DOTS OCR может потребоваться 2.x
- `pillow`: требуется <11.0.0, но установлена 12.1.0
- `transformers`: требуется <5.0.0, но установлена 5.0.0

## Решение: Отдельное виртуальное окружение

Рекомендуется создать отдельное виртуальное окружение для тестирования DOTS OCR:

```bash
# Создать новое виртуальное окружение
python -m venv venv_dots_ocr

# Активировать (Windows)
venv_dots_ocr\Scripts\activate

# Активировать (Linux/Mac)
source venv_dots_ocr/bin/activate

# Установить зависимости
pip install -r requirements_dots_ocr.txt

# Запустить скрипт
python test_dots_ocr.py
```

## Альтернативное решение: Использование Docker

Если конфликты зависимостей критичны, можно использовать Docker контейнер с изолированным окружением.

## Текущие конфликты

После установки пакетов обнаружены следующие конфликты:
- `numpy 2.4.1` vs требуемая `numpy<2.0.0`
- `pillow 12.1.0` vs требуемая `pillow<11.0.0`
- `transformers 5.0.0` vs требуемая `transformers<5.0.0`
- `torch 2.10.0` vs требуемая `torch==2.4.1` для torchvision

## FlashAttention2 (опционально)

**Важно:** `flash-attn` НЕ является обязательной зависимостью!

- Скрипт работает **без** flash-attn
- Flash-attn нужен только для ускорения на GPU
- Установка flash-attn на Windows часто проблематична (требует компиляции C++ кода)
- Если flash-attn не установлен, модель будет использовать стандартное внимание (медленнее, но работает)

### Если нужно установить flash-attn (только для GPU ускорения):

```bash
# Попробуйте установить (может не работать на Windows)
pip install flash-attn --no-build-isolation

# Или используйте предкомпилированные wheel файлы для вашей платформы
# См. https://github.com/Dao-AILab/flash-attention
```

**Рекомендация:** Для тестирования DOTS OCR flash-attn не нужен. Скрипт автоматически определит его наличие и будет использовать, если доступен.

## Примечание

Для продакшн использования DOTS OCR рекомендуется:
1. Использовать отдельный сервис/микросервис
2. Использовать Docker контейнер
3. Использовать API endpoint (если доступен)
