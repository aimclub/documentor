# Руководство по ручной разметке документов

## Зачем нужна ручная разметка?

Автоматическая разметка на основе нашего пайплайна создаст **нечестное преимущество** для нашего парсера:
- Наш парсер получит ~100 баллов (так как разметка создана по его выводу)
- Другие парсеры (Marker, Dedoc) получат низкие баллы из-за несоответствия форматов

**Ручная разметка** создает **объективный ground truth**, который справедливо оценивает все парсеры.

## Что мы размечаем?

### 1. Элементы документа

Размечаем **все значимые элементы** в порядке их появления:

- **Заголовки** (title, header_1 - header_6)
- **Текстовые блоки** (text)
- **Таблицы** (table)
- **Изображения** (image)
- **Списки** (list_item)
- **Подписи** (caption)
- **Ссылки** (link)
- **Формулы** (formula)
- **Код** (code_block)

### 2. Параметры каждого элемента

Для каждого элемента указываем:

#### Обязательные параметры:
- **`id`** - уникальный идентификатор (генерируется автоматически)
- **`type`** - тип элемента (header_1, text, table, etc.)
- **`content`** - текстовое содержимое элемента
- **`order`** - порядковый номер в документе (0, 1, 2, ...)

#### Важные для оценки:
- **`parent_id`** - ID родительского элемента (для иерархии)
- **`page_number`** - номер страницы (1-based)
- **`bbox`** - координаты [x0, y0, x1, y1] в пикселях или пунктах

#### Для таблиц дополнительно:
- **`metadata.table_structure.cells`** - массив ячеек с координатами (row, col, content)
- **`metadata.table_structure.html`** - HTML представление таблицы

## Как размечать?

### Шаг 1: Подготовка

1. Откройте PDF документ в PDF-просмотрщике
2. Откройте инструмент разметки
3. Загрузите документ в инструмент

### Шаг 2: Разметка элементов

Для каждого элемента в документе:

1. **Выделите элемент** в документе (клик или выделение области)
2. **Выберите тип** элемента из списка
3. **Скопируйте текст** элемента (если есть)
4. **Укажите координаты** (bbox) - автоматически или вручную
5. **Укажите номер страницы**
6. **Укажите родителя** (если элемент вложен в другой)

### Шаг 3: Проверка порядка

После разметки всех элементов:

1. Проверьте, что `order` соответствует порядку появления в документе
2. Проверьте иерархию - заголовки должны иметь правильные `parent_id`
3. Для таблиц проверьте структуру ячеек

### Шаг 4: Сохранение

Сохраните разметку в JSON файл согласно схеме `annotation_schema.json`.

## Важные принципы

### 1. Объективность
- Размечайте то, что **реально видно** в документе
- Не ориентируйтесь на вывод какого-либо парсера
- Используйте визуальное восприятие документа

### 2. Полнота
- Разметьте **все значимые элементы**
- Не пропускайте элементы, даже если они кажутся неважными
- Включите все таблицы, изображения, заголовки

### 3. Точность
- Текст должен быть **точным** (без опечаток)
- Координаты должны **соответствовать** реальному положению
- Порядок должен **соответствовать** порядку чтения

### 4. Иерархия
- Заголовки должны иметь правильную вложенность
- Текстовые блоки должны быть привязаны к ближайшему заголовку
- Таблицы и изображения должны быть привязаны к соответствующему разделу

## Метрики, которые мы оцениваем

Разметка должна позволить оценить:

1. **Element Detection** - все ли элементы найдены
2. **Ordering** - правильность порядка элементов
3. **Hierarchy** - правильность иерархической структуры
4. **Document TEDS** - структурное сходство всего документа
5. **Table TEDS** - структурное сходство таблиц

Поэтому важно размечать:
- ✅ Все элементы (для detection)
- ✅ Порядок (для ordering)
- ✅ Родителей (для hierarchy)
- ✅ Структуру таблиц (для table TEDS)
- ✅ Координаты (для дополнительной валидации)
