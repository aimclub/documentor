# Задача: Сбор метрик и разметка данных для оценки качества парсинга документов

## Цель задачи

Создать систему оценки качества парсинга документов, которая позволяет:
1. **Объективно сравнивать** различные парсеры (Documentor, Marker, Dedoc)
2. **Измерять метрики** качества парсинга по различным аспектам
3. **Выявлять слабые места** парсеров для улучшения

## Что мы оцениваем?

### 1. Обнаружение элементов (Element Detection)
**Метрики**: Precision, Recall, F1-Score

Оценивает, насколько точно парсер находит все элементы документа:
- Все ли элементы найдены (Recall)
- Все ли найденные элементы действительно есть в документе (Precision)
- Баланс между ними (F1-Score)

**Что размечаем**: Все элементы документа с их типами и содержимым.

### 2. Точность порядка (Ordering Accuracy)
**Метрика**: Процент элементов в правильном порядке

Оценивает, правильно ли парсер определяет порядок элементов в документе.

**Что размечаем**: Поле `order` для каждого элемента (0, 1, 2, ...) в порядке появления в документе.

### 3. Точность иерархии (Hierarchy Accuracy)
**Метрика**: Процент элементов с правильным parent_id

Оценивает, правильно ли парсер определяет иерархическую структуру документа (какие элементы вложены в какие заголовки).

**Что размечаем**: Поле `parent_id` для каждого элемента, указывающее на родительский заголовок или раздел.

### 4. Структурное сходство документа (Document TEDS)
**Метрика**: Tree-Edit-Distance-based Similarity для всего документа

Оценивает структурное сходство предсказанного парсинга с ground truth на основе дерева элементов.

**Что размечаем**: Полная структура элементов с иерархией (parent_id) и порядком (order).

### 5. Структурное сходство таблиц (Table TEDS)
**Метрика**: TEDS для каждой таблицы отдельно

Оценивает структурное сходство таблиц на основе их HTML структуры и содержимого ячеек.

**Что размечаем**: 
- HTML представление таблицы (`metadata.table_structure.html`)
- Структура ячеек с координатами (`metadata.table_structure.cells`)

### 6. Качество извлечения текста (Text Extraction Quality)
**Метрики**: BLEU, ROUGE, точность символов

Оценивает качество извлеченного текста для текстовых элементов.

**Что размечаем**: Точный текст каждого текстового элемента.

### 7. Качество извлечения таблиц (Table Extraction Quality)
**Метрики**: Точность HTML структуры, точность содержимого ячеек

Оценивает качество извлечения таблиц.

**Что размечаем**: HTML структура таблицы и содержимое каждой ячейки.

## Формат размеченных данных

### Общая структура

```json
{
  "document_id": "unique_document_id",
  "source_file": "path/to/source/document.pdf",
  "document_format": "pdf",  // "pdf", "docx", "pdf_scanned"
  "annotation_version": "2.0",  // Версия схемы разметки
  "annotator": "Имя разметчика",
  "annotation_date": "2024-01-15T10:30:00Z",
  "elements": [
    // Массив элементов в порядке появления
  ],
  "statistics": {
    "total_elements": 150,
    "total_pages": 10,
    "elements_by_type": {
      "title": 1,
      "header_1": 5,
      "header_2": 12,
      "text": 80,
      "table": 8,
      "image": 15,
      "formula": 20,
      "list_item": 9
    },
    "table_count": 8,
    "image_count": 15
  }
}
```

### Структура элемента

#### Базовые поля (обязательные)

```json
{
  "id": "elem_001",  // Уникальный идентификатор элемента
  "type": "header_1",  // Тип элемента (см. ниже)
  "content": "Заголовок раздела",  // Текстовое содержимое элемента
  "order": 0,  // Порядковый номер в документе (0-based)
  "parent_id": null  // ID родительского элемента (null для корневых)
}
```

#### Дополнительные поля (рекомендуемые)

```json
{
  "page_number": 1,  // Номер страницы (1-based)
  "bbox": [100, 50, 500, 80],  // Координаты [x0, y0, x1, y1] в пикселях
  "metadata": {
    // Дополнительные метаданные в зависимости от типа элемента
  }
}
```

### Типы элементов

Поддерживаемые типы элементов:

- **`title`** - Заголовок документа
- **`header_1`** - Заголовок уровня 1
- **`header_2`** - Заголовок уровня 2
- **`header_3`** - Заголовок уровня 3
- **`header_4`** - Заголовок уровня 4
- **`header_5`** - Заголовок уровня 5
- **`header_6`** - Заголовок уровня 6
- **`text`** - Текстовый блок
- **`table`** - Таблица
- **`image`** - Изображение
- **`formula`** - Математическая формула
- **`list_item`** - Элемент списка
- **`caption`** - Подпись к изображению/таблице
- **`footnote`** - Сноска
- **`page_header`** - Колонтитул (верхний)
- **`page_footer`** - Колонтитул (нижний)
- **`link`** - Ссылка
- **`code_block`** - Блок кода

### Специфичные форматы для разных типов элементов

#### 1. Текстовый элемент (text)

```json
{
  "id": "elem_002",
  "type": "text",
  "content": "Это текст параграфа. Он может содержать несколько предложений.",
  "order": 1,
  "parent_id": "elem_001",
  "page_number": 1,
  "bbox": [100, 90, 500, 200],
  "metadata": {
    "links": ["https://example.com"],  // Опционально: ссылки в тексте
    "font_size": 12,  // Опционально: размер шрифта
    "font_family": "Times New Roman"  // Опционально: семейство шрифта
  }
}
```

#### 2. Таблица (table)

**ВАЖНО**: Таблицы хранятся **только в HTML формате**, не в DataFrame.

```json
{
  "id": "elem_015",
  "type": "table",
  "content": "<table><thead><tr><th>Колонка 1</th><th>Колонка 2</th></tr></thead><tbody><tr><td>Значение 1</td><td>Значение 2</td></tr></tbody></table>",
  "order": 14,
  "parent_id": "elem_010",
  "page_number": 3,
  "bbox": [100, 300, 500, 450],
  "metadata": {
    "table_structure": {
      "html": "<table><thead><tr><th>Колонка 1</th><th>Колонка 2</th></tr></thead><tbody><tr><td>Значение 1</td><td>Значение 2</td></tr></tbody></table>",
      "cells": [
        {
          "row": 0,
          "col": 0,
          "content": "Колонка 1",
          "rowspan": 1,
          "colspan": 1
        },
        {
          "row": 0,
          "col": 1,
          "content": "Колонка 2",
          "rowspan": 1,
          "colspan": 1
        },
        {
          "row": 1,
          "col": 0,
          "content": "Значение 1",
          "rowspan": 1,
          "colspan": 1
        },
        {
          "row": 1,
          "col": 1,
          "content": "Значение 2",
          "rowspan": 1,
          "colspan": 1
        }
      ]
    },
    "rows_count": 2,
    "cols_count": 2,
    "caption": "Таблица 1: Результаты эксперимента"  // Опционально: подпись таблицы
  }
}
```

**Правила разметки таблиц**:
- `content` содержит **HTML строку** таблицы
- `metadata.table_structure.html` содержит тот же HTML (для удобства)
- `metadata.table_structure.cells` содержит массив всех ячеек с координатами (row, col)
- Для объединенных ячеек указываются `rowspan` и `colspan`
- Порядок ячеек в массиве не важен, важны координаты (row, col)

#### 3. Изображение (image)

**ВАЖНО**: Изображения хранятся **только в base64 формате**, не как локальные файлы.

```json
{
  "id": "elem_025",
  "type": "image",
  "content": "",  // Пустая строка для изображений
  "order": 24,
  "parent_id": "elem_020",
  "page_number": 5,
  "bbox": [150, 200, 400, 350],
  "metadata": {
    "image_data": "data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA...",  // Base64 encoded image
    "image_format": "png",  // Формат изображения
    "image_width": 250,  // Ширина в пикселях
    "image_height": 150,  // Высота в пикселях
    "caption": "Рисунок 1: Схема архитектуры"  // Опционально: подпись
  }
}
```

**Правила разметки изображений**:
- `content` всегда пустая строка для изображений
- `metadata.image_data` содержит base64 строку с префиксом `data:image/{format};base64,`
- Формат может быть: `png`, `jpeg`, `jpg`
- Рекомендуется использовать PNG для лучшего качества

#### 4. Формула (formula)

```json
{
  "id": "elem_030",
  "type": "formula",
  "content": "E = mc^2",  // LaTeX или текстовое представление формулы

  "metadata": {
    "latex": "E = mc^{2}",  // Опционально: LaTeX представление
    "formula_type": "inline"  // "inline" или "display"
  }
}
```

```json
{

  "metadata": {
    "level": 1,  // Уровень заголовка (1-6)

```

#### 6. Элемент списка (list_item)

```json
{

  "type": "list_item",
```

## Принципы разметки

### 1. Объективность
- Размечайте то, что **реально видно** в документе
- Не ориентируйтесь на вывод какого-либо парсера
- Используйте визуальное восприятие документа
- Разметка должна быть **независимой** от парсеров

### 2. Полнота
- Разметьте **все значимые элементы** документа
- Не пропускайте элементы, даже если они кажутся неважными
- Включите все таблицы, изображения, заголовки, формулы
- Учитывайте элементы на всех страницах

### 3. Точность
- Текст должен быть **точным** (без опечаток, с правильной пунктуацией)
- Координаты (bbox) должны **соответствовать** реальному положению элемента
- Порядок (order) должен **соответствовать** порядку чтения документа
- HTML таблиц должен быть **валидным** и точно отражать структуру

### 4. Иерархия
- Заголовки должны иметь правильную вложенность (header_1 > header_2 > header_3)
- Текстовые блоки должны быть привязаны к ближайшему заголовку выше
- Таблицы и изображения должны быть привязаны к соответствующему разделу
- Подписи (caption) должны быть привязаны к соответствующим элементам

### 5. Консистентность
- Используйте единый стиль разметки для всех документов
- Следуйте схеме `annotation_schema.json`
- Проверяйте разметку на валидность перед сохранением

## Процесс разметки

### Этап 1: Подготовка
1. Выберите документы для разметки (рекомендуется 5-10 документов разных типов)
2. Убедитесь, что документы репрезентативны (разные форматы, сложности)
3. Подготовьте инструмент разметки

### Этап 2: Первичная разметка
1. Откройте документ в инструменте разметки
2. Пройдитесь по документу страница за страницей
3. Для каждого элемента:
   - Определите тип элемента
   - Скопируйте/введите содержимое
   - Укажите координаты (bbox)
   - Укажите номер страницы
   - Укажите родительский элемент (parent_id)
   - Установите порядковый номер (order)

### Этап 3: Разметка таблиц
Для каждой таблицы:
1. Скопируйте HTML структуру таблицы (или создайте вручную)
2. Заполните массив `cells` с координатами каждой ячейки
3. Проверьте правильность rowspan/colspan для объединенных ячеек
4. Убедитесь, что HTML валидный

### Этап 4: Разметка изображений
Для каждого изображения:
1. Извлеките изображение из PDF
2. Конвертируйте в base64 формат
3. Сохраните в `metadata.image_data` с префиксом `data:image/{format};base64,`
4. Укажите размеры изображения (width, height)

### Этап 5: Проверка и валидация
1. Проверьте порядок элементов (order должен быть последовательным)
2. Проверьте иерархию (parent_id должны ссылаться на существующие элементы)
3. Проверьте полноту (все ли элементы разметены)
4. Проверьте валидность JSON
5. Проверьте статистику (соответствует ли она реальному документу)

### Этап 6: Сохранение
1. Сохраните разметку в JSON файл
2. Убедитесь, что файл соответствует схеме `annotation_schema.json`
3. Проверьте файл на валидность JSON

## Валидация разметки

После создания разметки проверьте её:

## Важные замечания

### Таблицы
- **Только HTML формат**: Таблицы хранятся только в HTML, не в DataFrame
- **Валидный HTML**: HTML должен быть валидным и правильно структурированным
- **Ячейки с координатами**: Массив `cells` должен содержать все ячейки с правильными координатами (row, col)

### Изображения
- **Только base64 формат**: Изображения хранятся только в base64, не как локальные файлы
- **Префикс data URL**: Используйте формат `data:image/{format};base64,{base64_string}`
- **Качество**: Рекомендуется использовать PNG для лучшего качества

### Иерархия
- **Корневые элементы**: Элементы верхнего уровня имеют `parent_id: null`
- **Вложенность заголовков**: header_2 должен быть вложен в header_1, header_3 в header_2 и т.д.
- **Привязка контента**: Текстовые блоки, таблицы, изображения должны быть привязаны к ближайшему заголовку выше

### Порядок
- **Последовательность**: order должен быть последовательным (0, 1, 2, 3, ...)
- **Порядок чтения**: order должен соответствовать порядку чтения документа (сверху вниз, слева направо)
