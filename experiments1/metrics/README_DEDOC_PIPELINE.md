# Пайплайн оценки метрик для Dedoc

## Что делает Dedoc?

Dedoc - это библиотека для автоматического парсинга документов. Она извлекает:
- Контент документа
- Логическую структуру (заголовки, параграфы, списки)
- Таблицы
- Метаинформацию

## Двухэтапный процесс

### Этап 1: Обработка документов через Dedoc

Сначала обрабатываем все PDF файлы через dedoc и сохраняем результаты:

```bash
cd experiments/metrics
python run_dedoc_pipeline.py
```

Этот скрипт:
1. Проверяет доступность Docker API dedoc
2. Обрабатывает все PDF файлы из `test_files_for_metrics/`
3. Сохраняет результаты в папку `dedoc_output/`:
   - `{document_id}_dedoc.json` - полный JSON результат
   - `{document_id}_dedoc.md` - извлеченный текст в Markdown

**Требования:**
- Docker контейнер dedoc должен быть запущен:
  ```bash
  docker run -d -p 1231:1231 --name dedoc dedocproject/dedoc
  ```

### Этап 2: Вычисление метрик

После обработки документов вычисляем метрики:

```bash
python evaluate_dedoc.py
```

Этот скрипт:
1. Автоматически обнаруживает сохраненные результаты в `dedoc_output/`
2. Загружает ground truth аннотации из `annotations/`
3. Вычисляет метрики:
   - CER (Character Error Rate)
   - WER (Word Error Rate)
   - TEDS (Tree-Edit-Distance-based Similarity)
   - Ordering accuracy
   - Hierarchy accuracy
4. Сохраняет результаты в `dedoc_metrics.json`

**Примечание:** Если сохраненные результаты не найдены, скрипт попытается обработать документы заново через Docker API или локальную установку.

## Структура файлов

```
experiments/metrics/
├── run_dedoc_pipeline.py      # Этап 1: Обработка документов
├── evaluate_dedoc.py           # Этап 2: Вычисление метрик
├── dedoc_output/               # Сохраненные результаты dedoc
│   ├── {document}_dedoc.json
│   └── {document}_dedoc.md
├── dedoc_metrics.json          # Итоговые метрики
└── test_files_for_metrics/     # Исходные PDF файлы
```

## Преимущества двухэтапного подхода

1. **Быстрое пересчитывание метрик**: Не нужно обрабатывать документы заново
2. **Анализ результатов**: Можно просмотреть JSON и Markdown файлы перед вычислением метрик
3. **Отладка**: Легче найти проблемы в обработке документов
4. **Гибкость**: Можно изменить логику вычисления метрик без переобработки документов
