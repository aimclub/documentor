# Эксперимент: Сравнение методов извлечения текста из PDF

## Описание

Эксперимент сравнивает два метода извлечения текста из PDF документов:

1. **Метод 1**: `prompt_layout_all_en` - Dots OCR извлекает layout и текст одновременно
2. **Метод 2**: `prompt_layout_only_en` + Qwen OCR - Dots OCR извлекает только layout, затем Qwen OCR извлекает текст

Оба метода сравниваются с ground truth (текст, извлеченный из PDF по координатам).

## Метрики

- **CER (Character Error Rate)**: Отношение неправильных символов к общему количеству символов
- **WER (Word Error Rate)**: Отношение неправильных слов к общему количеству слов (требует jiwer)

**Примечание**: Перед вычислением метрик текст нормализуется (удаление markdown форматирования, нормализация пробелов и переносов строк).

## Измерение времени

Для каждого метода измеряется:
- Время выполнения метода (в секундах и минутах)
- Время на один текстовый блок
- Общее время обработки файла
- Время извлечения ground truth
- Время вычисления метрик

Все времена сохраняются в JSON результаты и выводятся в консоль.

```

├── pipeline.py          # Основной пайплайн
```

## Установка зависимостей

```bash
pip install jiwer  # Для вычисления метрик CER/WER
```

## Использование

### Обработка одного файла

```bash
python pipeline.py test_files/journal-10-67-5-721-729.pdf
```

### Пакетная обработка всех файлов

```bash
python run_all.py
```

## Результаты

- Результатами обоих методов
- Метриками для каждого текстового блока
- Агрегированными метриками
- Ground truth данными
